## 1. 一句话总结

提出 RAPIER，一种在低质量训练数据（样本不足且标签含噪）下鲁棒的加密恶意流量检测系统，通过利用正常和恶意流量分布差异进行标签噪声校正和数据增强，在仅 1000 个样本和 45% 噪声率下，F1-score 达 0.770-0.855，相比现有方法平均提升 214.9%-352.6%。

## 2. 摘要翻译

**原文：**
Machine learning (ML) is promising in accurately detecting malicious flows in encrypted network traffic; however, it is challenging to collect a training dataset that contains a sufficient amount of encrypted malicious data with correct labels. When ML models are trained with low-quality training data, they suffer degraded performance. In this paper, we aim at addressing a real-world low-quality training dataset problem, namely, detecting encrypted malicious traffic generated by continuously evolving malware. We develop RAPIER that fully utilizes different distributions of normal and malicious traffic data in the feature space, where normal data is tightly distributed in a certain area and the malicious data is scattered over the entire feature space to augment training data for model training. RAPIER includes two pre-processing modules to convert traffic into feature vectors and correct label noises. We evaluate our system on two public datasets and one combined dataset. With 1000 samples and 45% noises from each dataset, our system achieves the F1 scores of 0.770, 0.776, and 0.855, respectively, achieving average improvements of 352.6%, 284.3%, and 214.9% over the existing methods, respectively. Furthermore, We evaluate RAPIER with a real-world dataset obtained from a security enterprise. RAPIER effectively achieves encrypted malicious traffic detection with the best F1 score of 0.773 and improves the F1 score of existing methods by an average of 272.5%.

**中文翻译：**
机器学习在准确检测加密网络流量中的恶意流方面具有前景；然而，收集包含足够数量加密恶意数据且标签正确的训练数据集具有挑战性。当 ML 模型使用低质量训练数据训练时，其性能会下降。本文旨在解决现实世界的低质量训练数据集问题，即检测由持续演变的恶意软件生成的加密恶意流量。我们开发了 RAPIER，充分利用特征空间中正常和恶意流量数据的不同分布——正常数据紧密分布在特定区域，而恶意数据分散在整个特征空间——来增强训练数据。RAPIER 包括两个预处理模块，将流量转换为特征向量并校正标签噪声。我们在两个公共数据集和一个组合数据集上评估系统。在每个数据集仅 1000 个样本和 45% 噪声下，系统分别达到 0.770、0.776 和 0.855 的 F1 分数，相比现有方法平均提升 352.6%、284.3% 和 214.9%。此外，我们使用从安全企业获取的真实数据集评估 RAPIER。RAPIER 有效地实现了加密恶意流量检测，最佳 F1 分数为 0.773，相比现有方法平均提升 272.5%。

## 3. 方法动机（Motivation）

### 3.1 问题背景与痛点

- **训练数据收集困难**：
  - 恶意软件持续演变，捕获的样本时间敏感性不足
  - 数据标签由恶意软件检测服务预测，不一定可靠（如 VirusTotal 的标签在不同年份可能变化）
  - 加密协议（如 SSL/TLS）阻止手动校正标签噪声
- **低质量数据的影响**：
  - 样本数量不足：仅 1000 个样本（500 恶意 + 500 正常）
  - 标签噪声率高：高达 45%
  - 现有方法在低质量数据上性能严重下降
- **现有方法的局限**：
  - 数据增强方法：在标签噪声下会生成更多噪声
  - 鲁棒 ML 模型：依赖强假设或大量训练数据
  - 迁移学习：需要额外大规模无标签数据，增加隐私泄露风险

### 3.2 核心直觉

- **分布差异观察**：正常数据分布紧密且一致，恶意数据分布稀疏且多样
- **标签噪声校正**：基于分布特征推断样本真实标签
- **数据增强**：预测新恶意数据的可能分布区域，合成新训练数据

## 4. 方法设计（Method Design）

### 4.1 整体流程

```
低质量训练数据 → 特征提取 → 标签噪声校正 → 数据增强 → 检测模型训练
   (1000样本,       (Auto-    (分布感知,     (GAN,      (Co-teaching
    45%噪声)        Encoder)   MADE)         分布预测)    + MLP)
```

### 4.2 三大模块

**模块一：特征提取（Feature Extraction）**
- **输入**：原始加密网络流量
- **处理**：
  - 基于改进的 Auto-Encoder 架构
  - 使用 bi-GRU 作为编码器和解码器
  - 以数据包长度序列为输入
  - 无监督训练，避免标签噪声污染特征
- **输出**：特征向量
- **关键点**：无监督方式提取特征，对标签噪声免疫

**模块二：标签噪声校正（Label Noise Correction）**
- **输入**：低质量训练数据的特征向量
- **处理**：
  - 使用 MADE（自回归生成模型）估计训练数据分布
  - 选择分布最密集区域的样本，重新标记为正常
  - 选择分布最稀疏区域的样本，重新标记为恶意
  - 使用集成学习（7 个经典 ML 分类器）推断其余样本的真实标签
- **输出**：标签校正后的训练数据
- **关键点**：利用正常和恶意数据的分布差异

**模块三：数据增强（Data Augmentation）**
- **输入**：标签校正后的训练数据
- **处理**：
  - 预测新恶意数据的可能分布区域
  - 使用改进的 GAN 模型合成新恶意训练数据
  - 合成新正常训练数据以维持决策边界
- **输出**：增强后的训练数据集
- **关键点**：生成位于目标分布区域的新数据，而非简单复制

**检测模型**：
- 使用 Co-teaching 和 MLP 构建检测器
- Co-teaching 消除残余标签噪声的影响
- 在增强后的数据和校正后的原始数据上训练

### 4.3 关键公式

- **Auto-Encoder 编码**：$f_{encoder} = (\overrightarrow{h}_n^{(1)}, \overleftarrow{h}_1^{(1)}, \dots, \overrightarrow{h}_n^{(B)}, \overleftarrow{h}_1^{(B)})$
- **MADE 分布估计**：$p(x) = \prod_{i=1}^{d} p(x_i | x_1 \dots x_{i-1})$
- **标签校正**：基于概率密度和欧氏距离的样本选择
- **GAN 数据增强**：生成器学习目标分布区域

### 4.4 优缺点

**优势：**
- 首个同时解决训练数据不足和标签噪声两个问题的系统
- 无监督特征提取对标签噪声免疫
- 基于分布的标签校正不依赖强假设
- GAN 数据增强生成多样化的训练数据
- 不需要额外数据，仅使用低质量训练集

**局限：**
- 假设正常数据分布紧密，恶意数据分布稀疏
- 不适用于恶意流量与正常流量分布完全相同的情况
- GAN 训练可能不稳定

## 5. 与其他方法对比（Comparison）

### 5.1 创新点

| 创新点 | 本文方法 | 现有方法 |
|--------|----------|----------|
| 数据质量问题 | 同时处理不足和噪声 | 仅处理一个问题 |
| 特征提取 | 无监督 Auto-Encoder | 监督或半监督 |
| 标签校正 | 基于分布（MADE） | 基于假设或损失值 |
| 数据增强 | GAN + 分布预测 | 过采样或标准 GAN |
| 额外数据需求 | 无 | 需要大规模无标签数据 |

### 5.2 与 Baseline 对比

与现有加密恶意流量检测方法对比：
- 传统 ML 方法
- 数据增强方法（SMOTE 等）
- 鲁棒 ML 方法（Co-teaching 等）
- 迁移学习方法

**关键差异**：
- 现有方法在低质量数据上性能严重下降，RAPIER 保持高性能
- 现有方法需要额外数据或强假设，RAPIER 仅需低质量训练集
- RAPIER 同时解决数据不足和标签噪声，现有方法仅处理一个问题

## 6. 实验表现（Experiments）

### 6.1 数据集

| 数据集 | 描述 | 规模 |
|--------|------|------|
| DoHBrw-2020 | DNS-over-HTTPS 流量 | 1000 样本 + 45% 噪声 |
| IDS2018 | 入侵检测数据集 | 1000 样本 + 45% 噪声 |
| AAGM-2017 | 恶意流量数据集 | 1000 样本 + 45% 噪声 |
| 组合数据集 | 上述三个数据集组合 | 1000 样本 + 45% 噪声 |
| 真实企业数据 | 安全企业提供的数据 | 真实场景 |

### 6.2 Baseline 方法

- 传统 ML 方法
- 数据增强方法（SMOTE、GAN 等）
- 鲁棒 ML 方法（Co-teaching、Decoupling 等）
- 迁移学习方法

### 6.3 评估指标

- F1-score
- Precision
- Recall
- 噪声率校正效果

### 6.4 关键结果

**低质量数据检测（1000 样本 + 45% 噪声）：**
- DoHBrw-2020：F1 = 0.770，提升 352.6%
- IDS2018：F1 = 0.776，提升 284.3%
- AAGM-2017：F1 = 0.855，提升 214.9%

**真实企业数据：**
- 最佳 F1 = 0.773
- 平均提升 272.5%

**标签噪声校正：**
- 在 20%-45% 噪声率下，将噪声率降低到 4.3% 以下
- 标签校正效果一致且稳定

**数据增强效果：**
- GAN 生成的数据有效提升检测性能
- 位于目标分布区域的数据比随机生成更有效

## 8. 总结（Summary）

### 8.1 核心思想

RAPIER 的核心思想是利用正常和恶意流量在特征空间中的分布差异，通过无监督特征提取避免标签噪声污染，基于分布的标签校正推断真实标签，GAN 数据增强生成位于目标分布区域的新数据。系统在仅 1000 个样本和 45% 噪声率下仍能有效检测加密恶意流量。

### 8.2 快速流程图

```
输入：低质量训练数据（1000 样本，45% 噪声）
  ↓
特征提取（无监督 Auto-Encoder，bi-GRU）
  ↓
标签噪声校正（MADE 分布估计 + 集成学习）
  ↓
数据增强（GAN + 分布预测）
  ↓
检测模型训练（Co-teaching + MLP）
  ↓
输出：加密恶意流量检测器
```
