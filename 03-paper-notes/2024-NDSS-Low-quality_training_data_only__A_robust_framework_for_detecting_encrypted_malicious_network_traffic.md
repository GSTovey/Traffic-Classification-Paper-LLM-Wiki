## 1. 一句话总结

提出 RAPIER，一种在低质量训练数据（样本不足且标签含噪）下鲁棒的加密恶意流量检测系统，通过利用正常和恶意流量分布差异进行标签噪声校正和数据增强，在仅 1000 个样本和 45% 噪声率下，F1-score 达 0.770-0.855，相比现有方法平均提升 214.9%-352.6%。

## 2. 摘要翻译

**原文：**
Machine learning (ML) is promising in accurately detecting malicious flows in encrypted network traffic; however, it is challenging to collect a training dataset that contains a sufficient amount of encrypted malicious data with correct labels. When ML models are trained with low-quality training data, they suffer degraded performance. In this paper, we aim at addressing a real-world low-quality training dataset problem, namely, detecting encrypted malicious traffic generated by continuously evolving malware. We develop RAPIER that fully utilizes different distributions of normal and malicious traffic data in the feature space, where normal data is tightly distributed in a certain area and the malicious data is scattered over the entire feature space to augment training data for model training. RAPIER includes two pre-processing modules to convert traffic into feature vectors and correct label noises. We evaluate our system on two public datasets and one combined dataset. With 1000 samples and 45% noises from each dataset, our system achieves the F1 scores of 0.770, 0.776, and 0.855, respectively, achieving average improvements of 352.6%, 284.3%, and 214.9% over the existing methods, respectively. Furthermore, We evaluate RAPIER with a real-world dataset obtained from a security enterprise. RAPIER effectively achieves encrypted malicious traffic detection with the best F1 score of 0.773 and improves the F1 score of existing methods by an average of 272.5%.

**中文翻译：**
机器学习在准确检测加密网络流量中的恶意流方面具有前景；然而，收集包含足够数量加密恶意数据且标签正确的训练数据集具有挑战性。当 ML 模型使用低质量训练数据训练时，其性能会下降。本文旨在解决现实世界的低质量训练数据集问题，即检测由持续演变的恶意软件生成的加密恶意流量。我们开发了 RAPIER，充分利用特征空间中正常和恶意流量数据的不同分布——正常数据紧密分布在特定区域，而恶意数据分散在整个特征空间——来增强训练数据。RAPIER 包括两个预处理模块，将流量转换为特征向量并校正标签噪声。我们在两个公共数据集和一个组合数据集上评估系统。在每个数据集仅 1000 个样本和 45% 噪声下，系统分别达到 0.770、0.776 和 0.855 的 F1 分数，相比现有方法平均提升 352.6%、284.3% 和 214.9%。此外，我们使用从安全企业获取的真实数据集评估 RAPIER。RAPIER 有效地实现了加密恶意流量检测，最佳 F1 分数为 0.773，相比现有方法平均提升 272.5%。

## 3. 方法动机（Motivation）

### 3.1 问题背景与痛点

- **训练数据收集困难**：
  - 恶意软件持续演变，捕获的样本时间敏感性不足
  - 数据标签由恶意软件检测服务预测，不一定可靠（如 VirusTotal 的标签在不同年份可能变化）
  - 加密协议（如 SSL/TLS）阻止手动校正标签噪声
- **低质量数据的影响**：
  - 样本数量不足：仅 1000 个样本（500 恶意 + 500 正常）
  - 标签噪声率高：高达 45%
  - 现有方法在低质量数据上性能严重下降
- **现有方法的局限**：
  - 数据增强方法：在标签噪声下会生成更多噪声
  - 鲁棒 ML 模型：依赖强假设或大量训练数据
  - 迁移学习：需要额外大规模无标签数据，增加隐私泄露风险

### 3.2 核心直觉

- **分布差异观察**：正常数据分布紧密且一致，恶意数据分布稀疏且多样
- **标签噪声校正**：基于分布特征推断样本真实标签
- **数据增强**：预测新恶意数据的可能分布区域，合成新训练数据

### §3.4 问题发现路径

## 4. 方法设计（Method Design）

### 4.1 整体流程

```
低质量训练数据 → 特征提取 → 标签噪声校正 → 数据增强 → 检测模型训练
   (1000样本,       (Auto-    (分布感知,     (GAN,      (Co-teaching
    45%噪声)        Encoder)   MADE)         分布预测)    + MLP)
```

### 4.2 三大模块

**模块一：特征提取（Feature Extraction）**
- **输入**：原始加密网络流量
- **处理**：
  - 基于改进的 Auto-Encoder 架构
  - 使用 bi-GRU 作为编码器和解码器
  - 以数据包长度序列为输入
  - 无监督训练，避免标签噪声污染特征
- **输出**：特征向量
- **关键点**：无监督方式提取特征，对标签噪声免疫

**模块二：标签噪声校正（Label Noise Correction）**
- **输入**：低质量训练数据的特征向量
- **处理**：
  - 使用 MADE（自回归生成模型）估计训练数据分布
  - 选择分布最密集区域的样本，重新标记为正常
  - 选择分布最稀疏区域的样本，重新标记为恶意
  - 使用集成学习（7 个经典 ML 分类器）推断其余样本的真实标签
- **输出**：标签校正后的训练数据
- **关键点**：利用正常和恶意数据的分布差异

**模块三：数据增强（Data Augmentation）**
- **输入**：标签校正后的训练数据
- **处理**：
  - 预测新恶意数据的可能分布区域
  - 使用改进的 GAN 模型合成新恶意训练数据
  - 合成新正常训练数据以维持决策边界
- **输出**：增强后的训练数据集
- **关键点**：生成位于目标分布区域的新数据，而非简单复制

**检测模型**：
- 使用 Co-teaching 和 MLP 构建检测器
- Co-teaching 消除残余标签噪声的影响
- 在增强后的数据和校正后的原始数据上训练

### §4.2 详细 Pipeline

| 步骤 | 输入 | 具体操作 | 输出 | 作用 |
|------|------|----------|------|------|
| Step 1 | 原始加密流量 | 五元组分割为 flow，取前 50 个包的长度序列 | 长度序列 l = [l₁,...,lₙ] | 数据预处理 |
| Step 2 | 长度序列 | Embedding 层映射为 V=32 维向量 | 嵌入序列 v = [v₁,...,vₙ] | 离散→连续 |
| Step 3 | 嵌入序列 | 2 层 bi-GRU 编码器，拼接最终隐状态 | 特征向量 f_encoder ∈ R^32 | 无监督特征提取 |
| Step 4 | 特征向量 | 2 层 bi-GRU 解码器 + MLP 重建层 | 重建序列 l̂ | 重建损失训练 AE |
| Step 5 | 训练集特征 | MADE 估计概率密度 p(x) | 每个样本的概率密度值 | 分布估计 |
| Step 6 | 概率密度值 | 选择高密度区样本重标为正常，低密度区重标为恶意 | N_s (正常), M_s (恶意) | 标签校正种子 |
| Step 7 | N_s + M_s | 7 个 ML 分类器集成学习推断其余标签 | 标签校正后的 D_train | 完整标签校正 |
| Step 8 | D_normal, D_malicious | 3 个生成器分别生成 M_B, M_O, N_B 区域数据 | 合成训练数据 | 数据增强 |
| Step 9 | 增强数据 + 校正数据 | Co-teaching + MLP 训练 | 检测器 | 最终模型 |

### §4.3 模型结构

| 模块 | 功能 | 输入 | 输出 | 与其他模块关系 |
|------|------|------|------|----------------|
| Embedding Layer | 包长度→32 维嵌入 | 长度序列 l | 嵌入序列 v | → Bi-GRU Encoder |
| Bi-GRU Encoder | 2 层双向 GRU 编码 | 嵌入序列 v | 特征向量 f (32 维) | → Decoder / MADE / 集成学习 |
| Bi-GRU Decoder | 2 层双向 GRU 解码 | 特征向量 f | 重建序列 l̂ | → 重建损失 |
| MADE | 自回归生成模型估计分布 | 特征向量集 | 概率密度 p(x) | → 标签校正 / 数据增强 |
| Ensemble (7 classifiers) | 集成学习推断标签 | N_s, M_s, 待校正样本 | 校正后标签 | → D_train |
| 3 Generators | 生成 M_B, M_O, N_B 区域数据 | 噪声 + 目标分布 | 合成数据 | → Discriminator |
| Discriminator | 区分原始/合成数据 | 原始 + 合成数据 | 二分类 | → 生成器对抗训练 |
| Co-teaching + MLP | 检测器 | 增强 + 校正数据 | 恶意/正常预测 | 最终输出 |

### §4.4 公式推导

**公式 1-3: Bi-GRU 特征提取**
$$h_t = GRU(h_{t-1}, v_t)$$
$$\overrightarrow{h}_t = \overrightarrow{GRU}(\overrightarrow{h}_{t-1}, v_t), \quad \overleftarrow{h}_t = \overleftarrow{GRU}(\overleftarrow{h}_{t+1}, v_t)$$
$$f_{encoder} = (\overrightarrow{h}_n^{(1)}, \overleftarrow{h}_1^{(1)}, \dots, \overrightarrow{h}_n^{(B)}, \overleftarrow{h}_1^{(B)}) \in \mathbb{R}^{2BH} = \mathbb{R}^{32}$$

- **设计选择**: 使用 bi-GRU 而非 CNN，因为包长度序列是时序数据；无监督训练避免标签噪声污染特征
- **维度选择**: d=2BH=2×2×8=32，平衡复杂度和表达能力

**公式 4: MADE 分布估计**
$$p(x) = \prod_{i=1}^{d} p(x_i | x_1 \dots x_{i-1}), \quad p(x_i | x_1 \dots x_{i-1}) = p_{\mathcal{M}}(x_i | \zeta_i)$$

- **推导**: MADE 将高维联合分布分解为条件概率的乘积（自回归分解），每个条件分布用高斯混合模型建模
- **优势**: 相比核密度估计，MADE 可处理高维数据；相比 VAE，MADE 直接输出概率密度而非隐变量
- **应用**: 仅用标记为正常的样本训练 MADE，避免恶意数据污染正常分布估计

**公式 5: 数据增强目标区域**
$$M_B = \{x | p_M(x) < \gamma, \omega_1 \leq p_N(x) < \omega_2\}$$
$$M_O = \{x | p_M(x) < \gamma, 0 \leq p_N(x) < \omega_1\}$$
$$N_B = \{x | p_M(x) < \gamma, \omega_2 \leq p_N(x) < \omega_3\}$$

- **M_B**: 接近正常数据边界的恶意数据区域（攻击者模仿正常行为）
- **M_O**: 远离所有已知数据的恶意数据区域（零日攻击）
- **N_B**: 正常数据边界内部（维持决策边界）

**公式 7: GAN 生成器 KL 散度损失**
$$\mathcal{L}_{KL(X_G||P)} = -\mathcal{H}(X_G) + \mathbb{E}_{x \in X_G, p_M(x) \geq \gamma}[\log p_M(x)] - \mathbb{E}_{x \in X_G, p_M(x)<\gamma, p_N(x)<\theta_1}[\log p_N(x)] + \mathbb{E}_{x \in X_G, p_M(x)<\gamma, p_N(x)\geq\theta_2}[\log p_N(x)]$$

- **推导**: 最小化生成分布与目标分布 P(x) 之间的 KL 散度，使生成数据落在目标区域
- **与标准 GAN 的区别**: 标准 GAN 学习训练数据分布，RAPIER 的 GAN 学习自定义目标分布

**公式 10: 判别器损失**
$$\mathcal{L}_D = \mathbb{E}_{D_{normal}}[\log \mathcal{D}(x)] + \mathbb{E}_{D_{malicious}}[\log(1-\mathcal{D}(x))] + \mathbb{E}_{G_{MB}}[\log(1-\mathcal{D}(x))] + \mathbb{E}_{G_{MO}}[\log(1-\mathcal{D}(x))] + \mathbb{E}_{G_{NB}}[\log \mathcal{D}(x)]$$

- **关键**: 判别器将 G_NB（合成正常数据）判为正常，将 G_MB、G_MO（合成恶意数据）判为恶意

### §4.5 方法优缺点

**优势：**
- 首个同时解决训练数据不足和标签噪声两个问题的系统
- 无监督特征提取对标签噪声免疫
- 基于分布的标签校正不依赖强假设
- GAN 数据增强生成多样化的训练数据
- 不需要额外数据，仅使用低质量训练集

**局限：**
- 假设正常数据分布紧密，恶意数据分布稀疏
- 不适用于恶意流量与正常流量分布完全相同的情况
- GAN 训练可能不稳定

## 5. 与其他方法对比（Comparison）

### 5.1 创新点

| 创新点 | 本文方法 | 现有方法 |
|--------|----------|----------|
| 数据质量问题 | 同时处理不足和噪声 | 仅处理一个问题 |
| 特征提取 | 无监督 Auto-Encoder | 监督或半监督 |
| 标签校正 | 基于分布（MADE） | 基于假设或损失值 |
| 数据增强 | GAN + 分布预测 | 过采样或标准 GAN |
| 额外数据需求 | 无 | 需要大规模无标签数据 |

### §5.2 创新点分析

| 创新点 | 说明 | 贡献度 | 是否可迁移 |
|--------|------|--------|-----------|
| 分布感知标签校正 | 利用正常/恶意数据分布差异，通过 MADE + 集成学习校正标签 | 高 | 是 |
| 目标区域数据增强 | 预测新恶意数据可能分布区域，3 个生成器分别生成 M_B/M_O/N_B 数据 | 高 | 是 |
| 无监督特征提取 | bi-GRU Auto-Encoder 无监督训练，对标签噪声免疫 | 高 | 是 |
| 同时解决不足+噪声 | 首个同时处理训练数据不足和标签噪声两个问题的系统 | 高 | 是 |
| 多 GAN 独立训练 | 训练 η=5 个独立 GAN 避免模型崩溃 | 中 | 是 |

## 7. 学习与应用（Learning & Application）

### 7.1 开源情况

- **代码**：论文未明确说明
- **可复现性**：提供了详细的算法描述和参数设置（Table II），但未明确提供代码

### §7.2 复现关键步骤

1. **数据预处理**: 五元组分割 flow，取前 50 个包的长度序列作为输入
2. **特征提取**: Embedding (V=32) → 2 层 bi-GRU 编码器 (H=8) → 拼接最终隐状态得到 32 维特征向量 → 2 层 bi-GRU 解码器重建 → 重建损失训练
3. **标签校正**: 用标记为正常的样本训练 MADE → 计算所有样本概率密度 → 选高密度区样本重标为正常 (N_s) → 选低密度区样本重标为恶意 (M_s) → 7 个 ML 分类器集成推断其余标签
4. **数据增强**: 用 D_normal 和 D_malicious 分别训练 MADE 得到 p_N(x) 和 p_M(x) → 定义 M_B/M_O/N_B 目标区域 → 训练 3 个生成器 + 1 个判别器 → 重复 η=5 次独立训练
5. **检测器训练**: Co-teaching + MLP 在增强数据 + 校正数据上训练

### §7.3 关键超参数

| 超参数 | 取值 | 说明 |
|--------|------|------|
| n (包数) | 50 | 每个流取前 50 个包 |
| V (嵌入维度) | 32 | Embedding 大小 |
| H (GRU 隐层) | 8 | 每层 GRU 隐状态维度 |
| B (GRU 层数) | 2 | bi-GRU 层数 |
| d (特征维度) | 2BH = 32 | 特征向量维度 |
| α (过滤比例) | 0.5 | MADE 后选择高密度样本的比例 |
| γ | 0.05 | 恶意概率密度第 5 百分位 |
| ω₁, ω₂, ω₃ | 0.1, 0.2, 0.3 | 正常概率密度第 10/20/30 百分位 |
| η (GAN 数量) | 5 | 独立 GAN 模型数量 |
| 学习率 | SGD | AE 训练优化器 |
| 数据划分 | T1 训练 / T2 测试 | 按时间戳划分 |

### §7.5 对研究的启发

1. **分布差异是标签校正的关键**: 正常数据紧密、恶意数据稀疏的假设在三个数据集上均成立，这一洞察可推广到其他异常检测领域
2. **无监督特征提取避免标签噪声传播**: AE 无监督训练是整个框架的基础，确保后续模块不被噪声标签污染
3. **目标区域增强优于简单过采样**: 不是复制已有数据，而是预测新数据可能出现的区域并生成，这对概念漂移场景尤为重要
4. **多 GAN 独立训练避免模式崩溃**: η=5 的策略简单有效，可迁移到其他 GAN 应用场景
5. **低维特征空间的重要性**: d=32 使 MADE 能准确估计分布，而 ETA 的 386 维特征导致 MADE 失效（F1=0），说明特征维度选择对下游模块有连锁影响

## 8. 总结（Summary）

### 8.1 核心思想

RAPIER 的核心思想是利用正常和恶意流量在特征空间中的分布差异，通过无监督特征提取避免标签噪声污染，基于分布的标签校正推断真实标签，GAN 数据增强生成位于目标分布区域的新数据。系统在仅 1000 个样本和 45% 噪声率下仍能有效检测加密恶意流量。

### 8.2 快速流程图

```
输入：低质量训练数据（1000 样本，45% 噪声）
  ↓
特征提取（无监督 Auto-Encoder，bi-GRU）
  ↓
标签噪声校正（MADE 分布估计 + 集成学习）
  ↓
数据增强（GAN + 分布预测）
  ↓
检测模型训练（Co-teaching + MLP）
  ↓
输出：加密恶意流量检测器
```
